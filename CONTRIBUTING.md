# Як редагувати контент сайту

Цей документ для довірених волонтерів, які керують контентом сайту Кіберкорпусу. Не потрібно знати Jekyll чи HTML — більшість змін робляться у звичайних текстових файлах.

---

## Що де редагувати

| Що змінити | Який файл відкрити |
|---|---|
| Головна сторінка (місія, цілі, кнопка Telegram) | `index.md` |
| Вкладка «СОВА» | `_includes/tab-beginners.md` |
| Вкладка «CDSS» | `_includes/tab-server.md` |
| Вступ до індивідуальних інструментів | `_includes/tab-individual-intro.md` |
| Посібник MHDDOS | `_includes/tab-mhddos.md` |
| Посібник DISTRESS | `_includes/tab-distress.md` |
| Посібник X100 | `_includes/tab-x100.md` |
| Топ-5 контрибюторів (таблиця лідерів) | `_data/contributors.yml` |
| Посилання на Telegram (глобально) | `_config.yml` → поле `telegram_url` |

---

## Редагування на GitHub (без git)

1. Відкрий потрібний файл у репозиторії на GitHub.
2. Натисни іконку олівця (Edit this file) у правому верхньому куті.
3. Внеси зміни.
4. Внизу сторінки натисни **Commit changes** → **Commit directly to the `main` branch**.
5. Сайт оновиться автоматично за ~1 хвилину.

## Форматування (Markdown)

Файли використовують стандартний Markdown:

```
## Заголовок другого рівня
### Заголовок третього рівня

Звичайний абзац тексту.

> Це цитата або важливе попередження.

**Жирний текст**, *курсив*.

- Пункт списку
- Ще один пункт

```bash
# Блок коду — для команд терміналу
sudo apt install something
```

## Структура файлів (для розуміння)

```
corp_site/
├── index.md                        ← Головна сторінка
├── tools.md                        ← Структура вкладок (не редагувати)
├── contributors.md                 ← Сторінка лідерів (не редагувати)
├── _includes/
│   ├── tab-beginners.md            ← Контент вкладки СОВА
│   ├── tab-server.md               ← Контент вкладки CDSS
│   ├── tab-individual-intro.md     ← Вступ до індивідуальних інструментів
│   ├── tab-mhddos.md               ← Посібник MHDDOS
│   ├── tab-distress.md             ← Посібник DISTRESS
│   └── tab-x100.md                 ← Посібник X100
├── _data/
│   └── contributors.yml            ← Дані таблиці лідерів
└── _config.yml                     ← Глобальні налаштування
```

Файли `tools.md` і `contributors.md` містять HTML-структуру — не редагуй їх без потреби. Весь змістовий контент винесено в `_includes/`.

## Якщо щось пішло не так

Якщо після коміту сайт не оновився або виглядає некоректно:

1. Відкрий вкладку **Actions** у репозиторії на GitHub.
2. Знайди останній запуск — там буде помилка із поясненням.
3. Найчастіша причина: порушено відступи у `.yml` файлі (YAML чутливий до пробілів).
